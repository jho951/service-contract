# Authz Audit Producer Contract

Authz 감사 계약은 `audit-log` 공통 모듈에 발행할 이벤트의 범위를 정의한다.
저장/보존/조회 원칙은 [Common Audit Contract](../../shared/audit.md)를 따른다.

## 이벤트 생성 시점
| 시점 | 설명 |
| --- | --- |
| 권한 허용 | 허용 이벤트 생성 |
| 권한 거부 | 거부 이벤트 생성 |
| 입력 오류 | 잘못된 요청 정보 기록 |
| 정책 조회 실패 | 정책 조회 실패 기록 |

## 감사 필드
| 필드 | 설명 |
| --- | --- |
| `requestId` | 요청 추적 ID |
| `correlationId` | 상관관계 추적 ID |
| `userId` | 사용자 식별자 |
| `userRole` | 사용자 역할 |
| `sessionId` | 세션 식별자 |
| `originalMethod` | 원본 메서드 |
| `originalPath` | 원본 경로 |
| `decision` | 최종 판정 |
| `reason` | 판정 사유 |

## decision 값
| 값 | 의미 |
| --- | --- |
| `ALLOW` | 허용 |
| `DENY` | 거부 |
| `BAD_REQUEST` | 입력 오류 |
| `ERROR` | 정책 조회/처리 오류 |

## 기록 원칙
| 원칙 | 설명 |
| --- | --- |
| 동시 기록 | 감사 이벤트는 관리 경로 판정과 동시에 남겨야 한다. |
| 추적 가능성 | 허용/거부와 감사 기록은 분리하되 요청 단위로 연결 가능해야 한다. |
| 신뢰 경계 | 외부 입력값은 그대로 저장하지 말고 Gateway 재주입 값과 서버 생성 추적 ID를 기준으로 기록한다. |

## 계약 원칙
| 원칙 | 설명 |
| --- | --- |
| 외부 노출 | 감사 API를 별도 외부 엔드포인트로 노출하지 않아도 된다. |
| 포맷 고정 | Authz는 공통 감사 포맷을 준수해 `audit-log`로 발행한다. |
| 저장 책임 | 저장/보존/조회는 `audit-log` 계약이 담당한다. |
